1  Vos obligations

Vous avez l’obligation pour les CCFs :

  • De déployer vos bases de données sur le serveur.
  • De déployer toutes vos applications web sur le serveur.

2  Identifiants de connexion

Vos identifiants de connexion sont initialement les mêmes sur tous les serveurs.

2.1  Login

Votre login est formé de la première lettre de votre prénom suivi de votre nom de famille, sans espace ni caractère spécial. Par exemple :

  • amesle pour Alexandre Meslé
  • aescriva pour Madame Escriva

2.2  Password

Votre password par défaut est azerty, il est très fortement conseillé de le modifier à la première connexion.

3  Serveur Web

Vous trouverez le serveur web à l’adresse 172.16.0.51. Pour déployer une application web, il vous suffit de placer les fichiers sources dans le répertoire ~/public_html.

3.1  Espace Web

Votre espace web est visible à l’URL 172.16.0.51/~login.

3.2  SSH

Connectez-vous une première fois en SSH pour modifier votre mot de passe.

  • Sous linux, utilisez la commande ssh login@172.16.0.51
  • Sous windows, utilisez le logiciel putty.

Le mot de passe se modifie avec la commande passwd.

3.3  Git

git est installé sur le serveur, il est fortement conseillé de l’utiliser pour déployer votre projet. Vous trouverez un tuto sur le site de Grafikart.

3.4  Filezilla

Vous pouvez aussi vous connecter à l’aide d’un client FTP en utilisant l’URL 172.16.0.51.

Achtung ! En général je désactive le service FTP. Si c’est le cas, utilisez WinScp.

3.5  Url rewriting

Attention, vos applications web vont être perturbées par les URL utilisées par apache. Adaptez et utilisez le fichier .htaccess suivant pour permettre l’URL rewriting :

RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^([\~a-zA-Z0-9]*)\/?$ /~votrelogin/index.php?p=$1 [QSA,L]
RewriteRule ^([a-zA-Z0-9]*)\/([0-9]{1,4})\/?$ /~votrelogin/index.php?p=$1&id=$2 [QSA,L]
RewriteRule ^([a-zA-Z0-9]*)\/([0-9]{1,4})\/([a-zA-Z]*)?$ /~votrelogin/index.php?p=$1&id=$2&action=$3 [QSA,L]

4  Serveur de base de données

Le serveur de bases de données se trouve à l’adresse 172.16.0.52.

4.1  phpmyadmin

Phpmyadmin se trouve à l’adresse http://172.16.0.52/phpmyadmin. Vos identifiants de connexion sont par défaut les mêmes.

N’oubliez pas de modifier le mot de passe !

  • —→Page d’accueil
  • —→Paramètres généraux
  • —→Modifier le mot de passe

4.2  Ligne de commande

Depuis votre ordinateur personnel ou le serveur web, vous pouvez vous connecter à mysql avec la commande mysql -u login -h 172.16.0.52 -p

4.3  Depuis une bibliothèque

Pour vous connecter depuis PDO ou JDBC :

  • host : 172.16.0.52
  • db : votre login
  • Mêmes login et password que pour accéder à phpmyadmin.

5  En cas de problème

En cas de problème technique, contactez Alexandre Meslé

5.1  Page "It Works!"

Vous vous êtes connecté au serveur 172.16.0.52 au lieu de 172.16.0.51.

5.2  Une seule base de données dans phpmyadmin

Vous ne pouvez voir qu’une seule base de données et vous n’avez pas les droits pour en créer une autre. A vous de préfixer vos tables pour permettre le déploiement de plusieurs applications dans la même base de données.

5.3  Je ne trouve pas le répertoire public_html

Vous devez le créer dans votre répertoire personnel avec la commande

mkdir ~/public_html
